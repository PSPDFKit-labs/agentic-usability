## Codebase Patterns
- Module resolution uses NodeNext — all local imports must end with `.js` extension
- Types are centralized in `src/core/types.ts` — import from there
- ESM project (`"type": "module"` in package.json) — use `import`, not `require`
- Quality checks: `npm run typecheck`, `npm run lint`, `npm run build`
