## Codebase Patterns
- Module resolution uses NodeNext — all local imports must end with `.js` extension
- Turndown is imported as `import TurndownService from 'turndown'` (default export)
- Types are centralized in `src/core/types.ts` — import from there
- ESM project (`"type": "module"` in package.json) — use `import`, not `require`
- Quality checks: `npm run typecheck`, `npm run lint`, `npm run build`
- Commands go in `src/commands/<name>.ts` and export an async function, wired in `src/index.ts`
- Use `node:readline/promises` for interactive CLI prompts (no extra dependency needed)
- Agent JSON output parsing: use `extractJson()` pattern to strip markdown fences and find array boundaries, then retry once on parse failure
- Shared suite validation and table printing lives in `src/commands/suite-utils.ts` — reuse for any command that reads/writes test suites
- cli-table3 imported as `import Table from 'cli-table3'` (default export)
- OpenSandbox SDK: `Sandbox.create()` handles readiness polling via `readyTimeoutSeconds`; `SandboxClient` in `src/sandbox/opensandbox.ts` wraps the SDK for simplified usage
- Docs fetcher: `fetchAndCacheDocs(publicInfo, options)` from `src/sandbox/docs-fetcher.ts` — caches in `.agentic-usability/cache/docs/` with 24h TTL, truncates to 100KB
- Workspace scaffolding: `scaffoldWorkspace(client, config, testCase)` from `src/sandbox/scaffolding.ts` — returns log string, layers are optional
- Results stored in `.agentic-usability/results/<test-id>/` — files: generated-solution.json, agent-output.log, setup.log, error.log
- Agent install: `getAgentInstallCommand()` in execute.ts maps known agents to npm install commands; custom agents skip install
- System prompt interpolation: `{{packageName}}` and `{{docsUrl}}` replaced from config.publicInfo
- WorkerPool: `new WorkerPool(concurrency)` from `src/sandbox/worker-pool.ts` — `run(testCases, executeFn, onProgress)` with retry (2 retries, 1s/3s backoff) and SIGINT graceful shutdown
- Token analysis: `analyzeTokens(solution, targetApis, expectedTokens, testId, target)` from `src/scoring/tokens.ts` — targetApis use word-boundary regex, expectedTokens support full regex
- Analyze command loads `generated-solution.json` from results dir, saves `token-analysis.json` per test case
- Judge scoring: `runJudge(testCase, generatedSolution, judgeConfig, target)` from `src/scoring/judge.ts` — returns JudgeScore, retries once on malformed JSON
- `formatSolution(files)` from `src/scoring/judge.ts` — renders SolutionFile[] as `--- File: path ---\ncontent` blocks
- Judge command saves `judge.json` per test case; errors saved as `judge-error.log`
